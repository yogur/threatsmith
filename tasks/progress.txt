## Codebase Patterns
- OWASP constants live in `prompts/references/owasp.py`; scanner snippet constants live in `prompts/references/scanner_snippets.py`. Do not import from the old top-level `prompts/owasp_references.py` or `prompts/scanner_snippets.py` — those files were removed in US-F03.

- `frameworks/_built_in.py` registers placeholder packs for all four frameworks at import time; `frameworks/__init__.py` imports it to trigger registration. Later pack builder stories will replace these placeholders.
- `list_frameworks()` returns `list(_REGISTRY.values())` — order follows insertion order (dict in Python 3.7+).

- Framework data model lives in `frameworks/models.py`; `frameworks/__init__.py` re-exports the public API. Keep `__init__.py` thin.
- `_REGISTRY` is a module-level dict in `models.py`; tests use the `autouse` fixture to snapshot/restore it so registrations don't leak between tests.
- `ruff` enforces E731 (no lambda assignments) — use `def` for named callables in tests.
