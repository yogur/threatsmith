## Codebase Patterns
- Framework data model lives in `frameworks/models.py`; `frameworks/__init__.py` re-exports the public API. Keep `__init__.py` thin.
- `_REGISTRY` is a module-level dict in `models.py`; tests use the `autouse` fixture to snapshot/restore it so registrations don't leak between tests.
- `ruff` enforces E731 (no lambda assignments) — use `def` for named callables in tests.
